# Firefox UX Writing Guidelines

Voice, tone, and writing rules for Firefox UI copy. Maintained by the [Firefox Content Design](https://acorn.firefox.com/latest/content/content-overview-ww9dIoVj) team.

---

## What's in this repo

These are the guidelines Firefox Content Design uses to write on-brand, consistent UI copy. Each file covers a specific area of writing practice.

| File | What it covers |
|------|----------------|
| `Brand-Voice-and-Tone.md` | Firefox's three voice attributes (Genuine, Fiery, Playful) and how to apply them |
| `Writing-Principles.md` | Core principles that apply to all Firefox copy |
| `Personas.md` | The three Firefox user personas and how to write for each |
| `Firefox-Word-List.md` | Preferred and prohibited terms for Firefox UI |
| `Capitalization.md` | Capitalization rules by surface and platform |
| `Punctuation.md` | Punctuation rules including em dashes, ellipses, and emojis |
| `Inclusive-Writing.md` | Guidelines for writing accessibly and inclusively |
| `Writing-for-Empty-States.md` | How to write titles, body copy, and actions for empty states |
| `Firefox-Story.md` | Firefox's mission and values, for use as a framing lens |
| `Felt-Privacy.md` | How to write about privacy and security features |
| `Conversion-Messaging.md` | Specific guidance for set default, sync, and import copy |
| `Voice-Tone-Examples.md` | Benchmark examples showing what great Firefox copy looks like |
| `Content-Evaluation-Criteria.md` | Rubric for evaluating UI copy across five dimensions |

## How to use these docs

These files are written to be read by both humans and AI tools. Each doc is a standalone reference — you can read one in isolation or use them together.

If you're using an AI coding or design tool, you can point it to the raw URL of any file in this repo to use it as a writing reference. Raw URLs follow this pattern:

```
https://raw.githubusercontent.com/ewachowiak/firefox-ux-writing/main/docs/FILENAME.md
```

For example, to reference the Word List:

```
https://raw.githubusercontent.com/ewachowiak/firefox-ux-writing/main/docs/Firefox-Word-List.md
```

## How to update or add a doc

All files in this repo must be `.md` (Markdown) files. Plain text files, Word docs, and PDFs are not accepted — they won't work correctly with the tools that read these guidelines.

**To update an existing file:**
1. Open the file on your computer and make your edits
2. Go to the `docs` folder in this repo on GitHub
3. Click **Add file > Upload files**
4. Drag in your updated file — GitHub will replace the old version
5. Add a short note describing what you changed (e.g. "Updated Word List: added tab groups entry")
6. Click **Commit changes**

**To add a new file:**
Follow the same steps above. Make sure the file is a `.md` file and that it's written clearly enough for both a person and an AI tool to understand — avoid internal shorthand, abbreviations, or references that wouldn't make sense to an outside reader.

**A note on file names:** use the same naming style as existing files (words separated by hyphens or spaces, no special characters). This keeps the file list consistent and URLs predictable.

---

## What not to add

- Files with confidential or internal-only information (internal metrics, research findings, unannounced features, campaign strategies)
- Non-Markdown file types (`.pdf`, `.docx`, `.txt`, `.csv`)
- Files that duplicate content already covered in an existing doc — update the existing file instead
